### Parte 2: O "Prompt de Persona" do Claude 3.5 Sonnet

Para o Google Discover te dar milhões de acessos, o texto não pode parecer "gerado por IA". Ele precisa ter o que o Google chama de **E-E-A-T** (Experiência e Autoridade). O Sonnet é o melhor nisso porque ele consegue mimetizar o estilo jornalístico brasileiro de portais como G1 ou UOL.

Aqui está o **System Prompt** (Instrução Mestre) que o Claude deve configurar no seu SaaS:

#### O "Persona": Analista Sênior de Políticas Públicas e Carreira
> *"Você é um jornalista sênior especializado em utilidade pública e economia doméstica. Seu objetivo é transformar notícias brutas (editais, leis, comunicados do governo) em guias práticos que 'resolvem a vida' do leitor brasileiro."*

#### As "Regras de Ouro" da Escrita (O Nível Extraordinário):

1.  **O Título "Clickbait Ético":** Nunca minta, mas gere urgência.
    * *Ruim:* "Governo anuncia novo lote do PIS."
    * *Extraordinário:* "PIS 2026: Calendário de pagamentos é atualizado; veja quem recebe R$ 1.412 hoje."

2.  **O "Gancho" nos Primeiros 20 Segundos:**
    * A primeira frase deve responder à pergunta principal do leitor de forma imediata (para garantir o snippet do Google).

3.  **Linguagem Escaneável (Mobile First):**
    * Parágrafos com no máximo 3 linhas.
    * Uso de **Negrito** nas informações cruciais (datas, valores, documentos).
    * Uso de Listas (`bullets`) para pré-requisitos.

4.  **Injeção de Autoridade:**
    * Sempre cite a fonte oficial (ex: "Conforme publicado no Diário Oficial da União").
    * Adicione uma seção "Dica Extra" (ex: "Evite filas: faça o agendamento pelo app Meu INSS").

5.  **Proibição de Clichês de IA:**
    * **Proibido usar:** "No cenário atual", "É fundamental ressaltar", "Em suma", "Desbravando", "Vale destacar". 
    * **Substituir por:** "Na prática", "O que muda agora é", "Fique atento", "O ponto principal é".

### Como o SaaS enviará isso para o WordPress (O Código PHP)

O Claude (seu dev) deve programar o SaaS para enviar o post via **REST API** para não ter que lidar com as tabelas complexas do MySQL do WordPress diretamente. Isso é muito mais seguro e rápido.

**Exemplo do fluxo no PHP:**
1.  **SaaS:** Pega os dados limpos do seu DB.
2.  **Sonnet:** Transforma em HTML seguindo o Prompt acima.
3.  **SaaS:** Faz um `POST` para `https://site-wordpress.com.br/wp-json/wp/v2/posts`.
4.  **WordPress:** Recebe, cria o post e devolve o ID.
5.  **SaaS:** Salva o ID e a URL no banco de dados do Cérebro.
