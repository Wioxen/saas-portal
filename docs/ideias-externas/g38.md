### 1.3 O Sistema de "Lock" de Processos (Anti-Colisão)
Como você terá **Cron Jobs** rodando a cada 5 ou 10 minutos, pode acontecer de um script de IA demorar mais que o previsto. Sem um "Lock", o sistema tentará rodar o mesmo processo duas vezes, gastando o dobro de tokens e possivelmente postando conteúdo duplicado.

* **Requisito para o Claude:** Implementar um sistema de `pid` (Process ID) ou um `status = 'processing'` no banco de dados. 
* **A lógica:** "Se existir um post com status 'processando', o script atual deve encerrar e esperar o próximo ciclo".

### 1.4 A Camada de "Abstração de API" (Payload Sanitizer)
A Anthropic e a OpenAI mudam seus formatos de JSON ocasionalmente. 
* **Requisito para o Claude:** Criar uma classe "Sanitizer" que limpa o texto retornado pela IA antes de salvar no banco. 
* **O motivo:** Muitas vezes o Claude ou GPT devolvem o texto com "Aqui está o seu post:" no início. O Sanitizer deve usar Regex para remover qualquer introdução da IA e entregar apenas o código HTML limpo.

### 1.5 Otimização de Logs (Debugging em Produção)
Quando você tem milhões de acessos, você não pode ficar lendo arquivos `.txt` de log gigantescos.
* **Requisito para o Claude:** A tabela `logs_sistema` deve ter uma coluna de `severidade` (INFO, WARNING, CRITICAL).
* **Ação:** Erros de API (falta de crédito ou timeout) devem ser `CRITICAL`. Notícias descartadas devem ser apenas `INFO`. Isso permitirá que você limpe os logs inúteis automaticamente e foque apenas no que impede o dinheiro de cair.
