Como você está usando o **Easypanel**, você tem a vantagem de rodar o **MariaDB** (que é o "irmão" mais performático e moderno do MySQL) em containers Docker. Para o seu SaaS PHP ser **extraordinário**, ele não pode misturar os dados dele com os dados do WordPress.

A arquitetura correta é: **O SaaS tem seu próprio Banco de Dados** que gerencia e "manda" nos 4 (ou 50) bancos de dados dos sites WordPress.

Aqui está o **esquema de banco de dados (Schema)** focado em performance, custos de API e rastreamento de lucro:

---

### 1. Tabela `ai_models` (Gestão de Orquestração)
Para o sistema saber qual IA usar e quanto ela custa.

```sql
CREATE TABLE ai_models (
    id INT AUTO_INCREMENT PRIMARY KEY,
    provider ENUM('anthropic', 'openai') NOT NULL,
    model_name VARCHAR(50) NOT NULL, -- ex: claude-3-5-sonnet, gpt-4o-mini
    input_cost_per_1k DECIMAL(10,5), -- Rastrear custos reais
    output_cost_per_1k DECIMAL(10,5),
    is_active BOOLEAN DEFAULT TRUE
);
```

### 2. Tabela `raw_trends` (A "Lixeira" de Triagem)
Onde o **GPT-4o-mini** faz a limpeza inicial.

```sql
CREATE TABLE raw_trends (
    id INT AUTO_INCREMENT PRIMARY KEY,
    source_url VARCHAR(500),
    raw_html LONGTEXT,
    extracted_data JSON, -- O JSON limpo gerado pelo GPT-4o-mini
    priority_score INT DEFAULT 0,
    status ENUM('pending', 'approved', 'rejected', 'failed') DEFAULT 'pending',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 3. Tabela `content_master` (O Cérebro da Operação)
Aqui registramos quem escreveu o quê e quanto custou. É aqui que você vê se está chegando nos **R$ 5.000,00**.

```sql
CREATE TABLE content_master (
    id INT AUTO_INCREMENT PRIMARY KEY,
    trend_id INT,
    site_id INT, -- Relacionado à sua tabela de sites WP
    title VARCHAR(255),
    final_content LONGTEXT,
    main_ai_model_id INT, -- ID do Sonnet
    triage_ai_model_id INT, -- ID do GPT-4o-mini
    input_tokens INT,
    output_tokens INT,
    estimated_cost_usd DECIMAL(10,5), -- Custo total da API para este post
    wp_post_id INT, -- ID gerado no site final
    status ENUM('draft', 'published', 'updated') DEFAULT 'draft',
    published_at TIMESTAMP NULL,
    FOREIGN KEY (main_ai_model_id) REFERENCES ai_models(id),
    FOREIGN KEY (trend_id) REFERENCES raw_trends(id)
);
```

### 4. Tabela `performance_logs` (O Termômetro de Lucro)
Para o nível extraordinário, o SaaS deve saber se o dinheiro gasto na API está voltando.

```sql
CREATE TABLE performance_logs (
    id INT AUTO_INCREMENT PRIMARY KEY,
    content_id INT,
    views_24h INT DEFAULT 0,
    clicks_count INT DEFAULT 0,
    ctr DECIMAL(5,2),
    revenue_est DECIMAL(10,2) DEFAULT 0.00, -- Cruzamento de dados do AdSense
    FOREIGN KEY (content_id) REFERENCES content_master(id)
);
```

---

### Por que essa estrutura é a melhor para o Claude e Easypanel?

1.  **JSON Native Support:** O MariaDB no Easypanel lida muito bem com o campo `extracted_data JSON`. Isso permite que o GPT-4o-mini guarde dados estruturados sem que você precise criar 50 colunas diferentes.
2.  **Relacionamentos Inteligentes:** Você consegue rodar uma query simples e descobrir: *"Quanto eu gastei com o Claude Sonnet no Site A e quanto ele me rendeu em cliques?"*.
3.  **Auditabilidade:** Se um post sair com erro, você sabe exatamente qual modelo de IA falhou e qual era o texto bruto original.

### Dica de Ouro para o Easypanel:
Como você vai rodar o MariaDB em um container, certifique-se de configurar o **Volume Persistence**. Se o container reiniciar e você não tiver volumes mapeados, você perde o histórico de custos e as tendências mineradas.

**Configuração no Easypanel:**
* **Service:** MariaDB
* **Mounts:** `/var/lib/mysql` -> `meu_saas_data`
