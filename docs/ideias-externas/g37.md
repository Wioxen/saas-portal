# 📑 Projeto: SaaS Maestro de Conteúdo Viral (Versão 1.0)
**Objetivo:** Automação total de rede de portais WordPress para dominação do Google Discover.
**Stack:** PHP 8.2+, MariaDB (Easypanel), Anthropic API (Sonnet 3.5), OpenAI API (4o-mini).

---

## 🏗️ FASE 1: Arquitetura de Dados (O Cérebro)

O sistema deve rodar de forma independente dos sites WordPress, gerenciando tudo via banco central.

### 1.1 Esquema MariaDB (Central SaaS)
Crie uma database `saas_core` com as seguintes tabelas críticas:

* **`sites_rede`**: Armazena `url_api`, `application_password`, `status` e `nicho`.
* **`raw_queue` (Triagem)**: Onde o scraper deposita HTML bruto.
* **`content_master`**: Registra o artigo final, `custo_tokens`, `wp_post_id` e métricas de performance.
* **`ai_config`**: Gerencia chaves de API e prioridade (Sonnet vs GPT).

### 1.2 Fluxo de Orquestração Híbrida
Implementar a lógica de roteamento de IA para economia e qualidade:
1.  **Triagem (GPT-4o-mini):** Lê o lixo do scraper -> Extrai fatos em JSON -> Descarta se for irrelevante.
2.  **Redação (Claude 3.5 Sonnet):** Recebe os fatos -> Escreve post em HTML com persona jornalística.
3.  **Refinamento (GPT-4o-mini):** Gera MetaTags, Schema FAQ e valida SEO.

---

## 🛠️ FASE 2: Motor de Produção de Elite

### 2.1 Persona de Escrita (Claude Sonnet 3.5)
O sistema deve injetar este **System Prompt** rigoroso:
* **Estilo:** Jornalismo de utilidade pública (Estilo G1/UOL).
* **Estrutura:** Parágrafos curtos (3 linhas), uso intensivo de tabelas HTML para dados, e **negrito** em termos-chave.
* **E-E-A-T:** Incluir sempre uma seção de "Dica do Especialista" para autoridade.
* **Anti-Bot:** Proibir terminologias comuns de IA (ex: "Em suma", "No cenário atual").

### 2.2 Processamento de Imagem (Imagick PHP)
Não apenas baixar imagens, mas processá-las:
* Aumentar Saturação (+12%) e Contraste (+5%).
* Converter para **WebP Progressivo**.
* Injetar texto dinâmico (Tarja de Urgência) via código.

---

## 🚀 FASE 3: Distribuição e Indexação de Guerrilha

### 3.1 Publicação Segura
* Conexão via **WordPress REST API**.
* Uso de **Application Passwords**.
* Header personalizado para identificar o SaaS no servidor.

### 3.2 Indexação em Milissegundos
* Integração com **Google Indexing API** (Service Account JSON).
* Envio imediato após a publicação.
* Logs de confirmação de indexação no banco de dados.

---

## 🔄 FASE 4: Ciclo de Vida e Atualização (O Diferencial)

### 4.1 O Módulo de "Shadow-Update"
O SaaS deve monitorar posts antigos e:
1.  Atualizar a data de publicação (Freshness).
2.  Reescrever a introdução para o contexto de "hoje".
3.  Limpar o cache do Cloudflare automaticamente.
