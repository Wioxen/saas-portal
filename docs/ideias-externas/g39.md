### Adições para a FASE 1 (A Fundação Blindada)

#### 1.6 O "Pool de Conexões" e Timeouts de API
* **O Problema:** A API da Anthropic (Sonnet) pode demorar até 30-40 segundos para gerar um post longo. Se o seu PHP não estiver configurado corretamente, a conexão "cai" e você paga pelo token, mas não recebe o texto.
* **Ação para o Claude:** Configurar o `GuzzleHttp` (ou cURL) com um `timeout` de pelo menos 60 segundos e implementar uma lógica de **Exponential Backoff** (se a API falhar, o sistema espera 2 segundos, depois 4, depois 8 antes de desistir).

#### 1.7 Sanitização de Banco de Dados (Auto-Cleanup)
* **O Problema:** O seu scraper vai coletar milhares de notícias. Em 3 meses, sua VPS de 2GB pode ficar sem espaço em disco por causa dos logs e HTMLs brutos.
* **Ação para o Claude:** Criar uma rotina que apaga automaticamente registros da tabela `raw_queue` que tenham mais de 15 dias e já foram processados ou rejeitados.

### Adições para a FASE 2 (A Engenharia do Conteúdo)

#### 2.5 O Módulo de "Link-E-Venda" (Affiliate Injector)
* **O Problema:** Depender só de AdSense é perigoso. O dinheiro de verdade está em CPA (Custo por Ação).
* **Ação para o Claude:** Criar uma tabela de "Keywords de Conversão". 
    * *Exemplo:* Sempre que o Sonnet escrever a palavra "Cartão de Crédito" ou "Empréstimo", o sistema deve injetar automaticamente um link de afiliado de um banco (ex: Banco Will, Inter, Nubank).
* **O Diferencial:** O link não deve ser estático. O SaaS deve poder trocar esse link em todos os 4 sites de uma vez só, direto pelo painel do Cérebro.

#### 2.6 Detecção de "Alucinação" da IA
* **O Problema:** Às vezes a IA inventa datas ou valores (alucinação).
* **Ação para o Claude:** Implementar uma regra de **Validação de Fatos Críticos**. 
    * O GPT-4o-mini (na fase de triagem) deve extrair valores numéricos e datas. Se o texto final do Sonnet não contiver esses mesmos números, o post entra em "Revisão Manual". Isso evita processos judiciais ou punições do Google por desinformação.

## 🛡️ FASE 3.3: Blindagem de Comunicação (IP Whitelisting)
Claude, para que ninguém consiga "hackear" seus sites fingindo ser o SaaS:
* **Validação de IP:** No WordPress (via `functions.php`), crie um filtro que bloqueia qualquer requisição para a REST API (`/wp-json/`) que não venha do IP fixo da sua VPS do Easypanel.
* **Secret Key Header:** O SaaS deve enviar um Header customizado (ex: `X-Custom-Auth-Key: [CHAVE_RANDOM]`). Se o WordPress receber a requisição sem essa chave, ele descarta o acesso.
