### 1. Arquitetura de Banco de Dados (O Cérebro Organizacional)
Não deixe o Claude criar tabelas genéricas. Para escalar para 50 sites, a estrutura de dados precisa ser **Multi-Tenant** (ou preparada para isso).

* **Tabela `trends_radar`**: Deve armazenar o `trend_source` (DOU, Google Trends, X), o `momentum_score` (velocidade com que a busca cresce) e o `status` (pendente, em IA, publicado).
* **Tabela `sites_inventory`**: Armazena as credenciais de cada site (URL, API Key do WordPress, ID do AdSense, Categoria padrão).
* **Tabela `content_lifecycle`**: Esta é a mais importante. Ela registra quando o post foi criado, quantas vezes foi atualizado e o `last_click_count`. É aqui que o módulo de atualização decide agir.

### 2. A Lógica de "Prompt Chaining" (Engenharia de Conteúdo)
O Claude precisa programar o SaaS para não fazer apenas uma chamada de IA, mas uma **corrente de raciocínio**:

1.  **Chamada 1 (Pesquisador)**: "Extraia os dados técnicos, datas e valores desta notícia bruta."
2.  **Chamada 2 (Redator)**: "Com base nos dados da Chamada 1, escreva um post usando a técnica de *Inverted Pyramid* (informação mais importante primeiro) para o site [NOME]."
3.  **Chamada 3 (SEO Expert)**: "Crie 3 variações de títulos (H1) que gerem curiosidade extrema e defina o *Alt Text* da imagem."

### 3. Integração de APIs de Terceiros (O Ecossistema)
Para o SaaS ser completo, o Claude deve implementar módulos de conexão com:

* **Google Indexing API**: Para indexação em segundos.
* **Search Console API**: Para o SaaS saber quais posts estão perdendo posição e precisam de atualização.
* **API de Imagens (DALL-E 3 ou Midjourney)** ou o script **Imagick** que detalhamos, para que o SaaS nunca use imagens do Google.
* **OneSignal ou Pushengage API**: Para disparar notificações automáticas no segundo em que o post vai ao ar.

---

### 4. O "Módulo de Blindagem" (Anti-Penalização)
Peça ao Claude para codificar regras de **Segurança Algorítmica**:

* **Spinning Inteligente**: Se uma notícia for postada em 2 sites da rede, a IA deve obrigatoriamente reescrever o texto com um tom de voz diferente (Ex: Site 1 é "Informativo", Site 2 é "Crítico/Opinativo").
* **Randomização de Delay**: O SaaS nunca deve postar em intervalos fixos (ex: de 10 em 10 minutos). Ele deve "imitar o humano", postando às 07:12, depois às 07:45, depois às 08:05.

### 5. O Dashboard de "Heatmap de Lucro"
Sua interface administrativa em PHP deve mostrar:
* **Lucro Real Estimado**: Cruzando visualizações (via Analytics) com o CPC médio do nicho.
* **Sugestão de "Trending Now"**: O sistema avisa: "O post sobre 'Auxílio' está bombando. Deseja criar 3 variações agora?".

---

### 6. Configuração de Deploy (Easypanel/Docker)
Instrua o Claude a gerar um arquivo `docker-compose.yml` que inclua:
* **O container PHP-FPM** (O SaaS).
* **O container Redis** (Para filas de tarefas e cache de trends).
* **O container de Workers** (Scripts que rodam em background 24/7 processando as notícias).
